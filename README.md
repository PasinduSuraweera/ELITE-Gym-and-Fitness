## 🎯 Overview

**ELITE Gym & Fitness** is a comprehensive, full-stack fitness management platform built specifically for the Sri Lankan fitness market. It combines modern web technologies with AI-powered features to deliver an exceptional user experience for gym members, trainers, and administrators.

### 🌟 What Makes Us Special

- 🤖 **AI-Powered Fitness Programs** - Personalized workout and diet plans using Google Gemini AI
- 💳 **Local Payment Integration** - Stripe payments optimized for Sri Lankan Rupees (LKR)
- 📱 **Mobile-First Design** - Responsive interface optimized for all devices
- 🎯 **Role-Based Access** - Separate dashboards for users, trainers, and admins
- 🛒 **E-Commerce Integration** - Full marketplace for fitness products and supplements
- 📊 **Real-Time Analytics** - Live tracking of memberships, bookings, and sales
- 🌙 **Dark Mode First** - Beautiful dark theme as default with light mode option

## 🚀 Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:
- **Node.js** 20.x or higher
- **npm** or **yarn**
- **Git**

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/PasinduSuraweera/ELITE-Gym-and-Fitness.git
cd ELITE-Gym-and-Fitness
```

2. **Install dependencies**
```bash
npm install
# or
yarn install
```

3. **Set up environment variables**

Create a `.env.local` file in the root directory:

```env
# Convex
CONVEX_DEPLOYMENT=your_convex_deployment_url
NEXT_PUBLIC_CONVEX_URL=your_convex_url

# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
CLERK_SECRET_KEY=your_clerk_secret_key
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up

# Stripe
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_stripe_webhook_secret

# Google Gemini AI
GEMINI_API_KEY=your_gemini_api_key

# Vapi AI (Optional)
NEXT_PUBLIC_VAPI_PUBLIC_KEY=your_vapi_public_key

# Chatbase (Optional)
NEXT_PUBLIC_CHATBASE_ID=your_chatbase_id
```

4. **Set up Convex**
```bash
npx convex dev
```

5. **Run the development server**
```bash
npm run dev
# or
yarn dev
```

6. **Open your browser**

Navigate to [http://localhost:3000](http://localhost:3000)

### First-Time Setup

1. **Create Admin Account**
   - Sign up through the application
   - Manually set role to "admin" in Convex dashboard

2. **Configure Stripe Products**
   - Run the price ID script: `node get-price-ids.js`
   - Update membership plans with your Stripe product IDs

3. **Seed Sample Data** (Optional)
   - Navigate to `/admin/seed-recipes` to populate recipe database
   - Use the admin panel to add initial blog posts and products

## ⚙️ Configuration

### Convex Schema Overview

The database uses Convex for real-time data management with the following main tables:

- **users** - User profiles with role-based access (user/trainer/admin)
- **memberships** - Active and expired memberships with tier information
- **bookings** - Trainer session bookings with status tracking
- **cart** - Shopping cart items for marketplace
- **orders** - Order history with payment and shipping details
- **products** - Marketplace inventory
- **blog** - Blog posts with categories and engagement metrics
- **recipes** - Recipe database with nutritional information
- **reviews** - Product reviews and ratings
- **trainerProfiles** - Trainer information and availability
- **salary** - Trainer payment records

### Stripe Configuration

Three main Stripe product types:
1. **Memberships** - Monthly recurring subscriptions
2. **Trainer Bookings** - One-time session payments
3. **Marketplace** - Product purchases with checkout

### Email & Phone Validation

The platform includes comprehensive form validation:
- **Email**: RFC 5322 compliant with domain TLD validation
- **Phone**: Sri Lankan format validation (10 digits, valid prefixes)

## 🌐 Deployment

### Deploy to Vercel

1. **Push to GitHub**
```bash
git add .
git commit -m "Initial commit"
git push origin main
```

2. **Import to Vercel**
   - Go to [vercel.com](https://vercel.com)
   - Click "New Project"
   - Import your GitHub repository
   - Add environment variables
   - Deploy!

3. **Configure Webhooks**
   - Update Stripe webhook URL to your production domain
   - Test webhook delivery

### Environment Variables for Production

Ensure all environment variables are set in Vercel:
- Convex URLs (production deployment)
- Clerk keys (production instance)
- Stripe keys (live mode)
- Google Gemini API key
- Optional: Vapi, Chatbase keys

## 📚 API Documentation

### Convex Functions

#### User Management
```typescript
// Get user profile
api.users.getUserProfile({ clerkId: string })

// Update user role
api.users.updateUserRole({ clerkId: string, role: "user" | "trainer" | "admin" })
```

#### Memberships
```typescript
// Create membership
api.memberships.createMembership({
  clerkId: string,
  tier: "basic" | "premium" | "elite",
  duration: number,
  amount: number
})

// Check active membership
api.memberships.getActiveMembership({ clerkId: string })
```

#### Bookings
```typescript
// Create booking
api.bookings.createBooking({
  clerkId: string,
  trainerId: string,
  date: string,
  timeSlot: string,
  sessionType: string
})

// Get user bookings
api.bookings.getUserBookings({ clerkId: string })
```

#### Marketplace
```typescript
// Add to cart
api.cart.addToCart({
  clerkId: string,
  productId: string,
  quantity: number
})

// Create order
api.orders.createOrder({
  clerkId: string,
  items: array,
  shippingAddress: object,
  totalAmount: number
})
```

### REST API Endpoints

#### Stripe Checkout
```
POST /api/create-checkout-session
POST /api/create-marketplace-checkout
POST /api/create-booking-checkout
POST /api/webhooks/stripe
```

```typescript

  primary: "hsl(0, 84%, 50%)",  // Red theme
```
